## 一、系统概述

本系统基于 ESP32-S3 实现一个示波器前面板控制器：

- 4 路旋钮编码器分别调节 CH1/CH2 的时基与电压档；
- 4 个按键：**RUN** 切换内置波形/ADC 实采，**TDR** 切换 TDR 模式，**CH1/CH2** 分别控制各自波形的显示/隐藏；
- 4 颗指示灯：RUN/TDR 对应模式指示，CH1/CH2 对应波形可见性指示（**亮=显示，灭=隐藏**）；
- ST7789 LCD 实时渲染双通道波形 + 顶部信息栏（时基 / 电压档 / 模式）；
- UART 与 WiFi TCP 同步上报全部编码器、按键、模式及状态帧。

```
┌─────────────────────────────────────────────┐
│  EC11 × 4         ESP32-S3           LCD    │
│  ENC1 CH1-H  ──→  ┌─────────┐  ←── ST7789  │
│  ENC2 CH1-V  ──→  │         │       240×240 │
│  ENC3 CH2-H  ──→  │  panel  │──→ UART 上位机│
│  ENC4 CH2-V  ──→  │  state  │              │
│                   │         │──→ WiFi 日志  │
│  BTN × 4     ──→  └─────────┘              │
│  LED × 4     ←──                           │
└─────────────────────────────────────────────┘
```

> **开机默认**：显示内置测试波形（CH1 正弦 + CH2 三角），CH1/CH2 均可见，RUN/TDR 灭。

### 2.2 EC11 接线

```
EC11 引脚      ESP32-S3
  A  ─────────  GPIOx（ENCn_PIN_A）
```

## 三、LCD 显示界面

LCD 全屏 240×240，布局如下：

```
┌────────────────────────────┐  ← y=0
│  1H:100us  V:1V      INT  │  第1行（y=0）: CH1 时基+电压 / 模式
│  2H:100us  V:1V           │  第2行（y=8）: CH2 时基+电压
├────────────────────────────┤  ← y=16
│  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  │
│     ╔══════╗               │  CH1 绿色正弦
│    ╱╲    ╱╲    ╱╲          │  CH2 青色三角
│   ╱  ╲  ╱  ╲  ╱  ╲         │
│  ╱    ╲╱    ╲╱    ╲        │
│                            │
│  8行 × 10列 刻度网格（224px）│
│  暗灰色网格线，中心线较亮   │
│                            │
└────────────────────────────┘  ← y=240
```

- **CH1**：绿色正弦波，振幅 ≈ 2.8 格（内置测试模式下）
- **CH2**：青色三角波，振幅 ≈ 2.0 格（内置测试模式下）
- 旋转 ENC1 / ENC3 改变对应通道的时基（频率）
- 旋转 ENC2 / ENC4 改变对应通道的电压档
- 按 BTN_CH1 / BTN_CH2 切换各自波形的显隐，对应 LED 同步开/关
- 信息栏两行显示：
  - 第1行（y=0）：`1H:<时基> V:<电压>`（CH1）+ 模式标签（右对齐）
  - 第2行（y=8）：`2H:<时基> V:<电压>`（CH2）
  - 模式标签：`INT` = 内置波形（默认）、`RUN` = ADC 实采、`TDR` = TDR 模式
  - 隐藏通道对应行文字变为暗灰色

> 开机默认 `WAVE_SRC_SINE`（内置 CH1 正弦 + CH2 三角）；按 RUN 后自动切换至 `WAVE_SRC_ADC` 显示真实采样波形。

## 四、操作说明

### 4.1 旋钮编码器

| 编码器 | 位置 | 功能 | 效果 |
|--------|------|------|------|
| ENC1 | CH1 左旋钮 | 时基（水平）| 顺时针→档位增大（更慢），逆时针→档位减小（更快），共 29 档：2ns ~ 5s |
| ENC2 | CH1 右旋钮 | 电压档（垂直）| 顺时针→档位增大（更大量程），逆时针→减小，共 13 档：1mV ~ 10V |
| ENC3 | CH2 左旋钮 | 时基（水平）| 同 ENC1，控制 CH2 |
| ENC4 | CH2 右旋钮 | 电压档（垂直）| 同 ENC2，控制 CH2 |

时基档位（29 档）：
```
2ns 5ns 10ns 20ns 50ns 100ns 200ns 500ns
1us 2us 5us 10us 20us 50us 100us 200us 500us
1ms 2ms 5ms 10ms 20ms 50ms 100ms 200ms 500ms
1s 2s 5s
```

电压档位（13 档）：
```
1mV 2mV 5mV 10mV 20mV 50mV 100mV 200mV 500mV 1V 2V 5V 10V
```

### 4.2 按键功能

| 按键 | GPIO | 功能 | 说明 |
|------|------|------|------|
| BTN_RUN | GPIO9  | 切换 内置波形 ↔ ADC 实采 | 当前为 MODE_RUN 则切回 MODE_INTERNAL，否则切至 MODE_RUN |
| BTN_CH1 | GPIO10 | CH1 显示/隐藏切换        | 每按一次 CH1 波形切换显隐；LED_CH1 随之亮/灭 |
| BTN_TDR | GPIO11 | 切换 TDR 模式            | 当前为 MODE_TDR 则切回 MODE_INTERNAL，否则切至 MODE_TDR |
| BTN_CH2 | GPIO12 | CH2 显示/隐藏切换        | 每按一次 CH2 波形切换显隐；LED_CH2 随之亮/灭 |

> RUN 与 TDR 互斥：每次只有一个处于激活状态；再次按下对应按键即返回 MODE_INTERNAL。

### 4.3 指示灯状态

| LED | 颜色 | 亮起条件 |
|-----|------|---------|
| LED_RUN | 绿 | `MODE_RUN`（ADC 实采模式） |
| LED_TDR | 黄 | `MODE_TDR`（TDR 模式） |
| LED_CH1 | 蓝 | CH1 可见 (`ch[0].visible == 1`) |
| LED_CH2 | 白 | CH2 可见 (`ch[1].visible == 1`) |

> 开机默认：LED_CH1 + LED_CH2 亮（两通道可见），LED_RUN + LED_TDR 灭（内置波形）。

## 五、WiFi 日志查看

系统上电后自动启动 WiFi SoftAP，所有 `ESP_LOGI/W/E` 日志同时输出至串口和 WiFi TCP 客户端。

### 5.1 连接步骤

1. 在手机/电脑的 WiFi 列表中找到 **`timer`**，点击连接（无密码）
2. 连接成功后设备 IP 为 `192.168.4.2`，ESP32-S3 的 IP 为 `192.168.4.1`

### 5.2 查看日志

**Python（推荐）：**
```python
import socket

s = socket.socket()
s.connect(('192.168.4.1', 4444))
print(s.recv(256).decode())  # 欢迎信息
while True:
    data = s.recv(4096)
    if not data:
        break
    print(data.decode(), end='')
```

**命令行（Linux/macOS）：**
```bash
nc 192.168.4.1 4444
```

**命令行（Windows PowerShell）：**
```powershell
$s = New-Object Net.Sockets.TcpClient('192.168.4.1', 4444)
$r = New-Object IO.StreamReader($s.GetStream())
while($true){ Write-Host $r.ReadLine() }
```

> 最多支持 4 路 TCP 同时连接。断开重连后日志继续输出，不会影响系统运行。

## 六、UART 通信协议

UART1，115200bps，8N1，TX=GPIO17，RX=GPIO18。

### 帧格式

```
[0xAA][0x55][TYPE][LEN][DATA...][CHECKSUM]
```

- `CHECKSUM = TYPE ^ LEN ^ DATA[0] ^ DATA[1] ^ ...`

### 事件类型

| TYPE | 名称 | DATA 格式 | 触发条件 |
|------|------|-----------|---------|
| 0x01 | ENCODER_EVENT | `[enc_id][dir]` | 每次旋钮转动 |
| 0x02 | BUTTON_EVENT  | `[btn_id][state]` | 每次按键状态变化 |
| 0x03 | MODE_CHANGE   | `[mode]` | 按键触发模式切换 |
| 0x04 | STATUS_REPORT | `[mode][ch1_h][ch1_v][ch1_vis][ch2_h][ch2_v][ch2_vis]` | 每秒心跳时 |
| 0x05 | HEARTBEAT     | （空）| 每秒定时 |

### 枚举值

**enc_id：** 0=CH1-H，1=CH1-V，2=CH2-H，3=CH2-V  
**dir：** 0x01=顺时针，0xFF=逆时针  
**btn_id：** 0=RUN，1=CH1，2=TDR，3=CH2  
**state：** 0x01=按下，0x00=释放  
**mode：** 0=INTERNAL（内置波形），1=RUN（ADC 实采），2=TDR  
**chX_vis：** 0=隐藏，1=显示  

### 解帧示例（Python）

```python
import serial

ser = serial.Serial('COMX', 115200, timeout=1)

def read_frame(ser):
    while True:
        if ser.read(1) == b'\xaa' and ser.read(1) == b'\x55':
            ftype = ser.read(1)[0]
            flen  = ser.read(1)[0]
            data  = ser.read(flen)
            csum  = ser.read(1)[0]
            calc  = ftype ^ flen
            for b in data:
                calc ^= b
            if calc == csum:
                return ftype, data

while True:
    t, d = read_frame(ser)
    if t == 0x01:
        enc_names = ['CH1-H', 'CH1-V', 'CH2-H', 'CH2-V']
        print(f'ENC {enc_names[d[0]]}: {"CW" if d[1]==1 else "CCW"}')
    elif t == 0x02:
        btn_names = ['RUN','CH1','TDR','CH2']
        print(f'BTN {btn_names[d[0]]}: {"PRESS" if d[1]==1 else "RELEASE"}')
    elif t == 0x04:
        modes = ['INTERNAL','RUN','TDR']
        print(f'STATUS mode={modes[d[0]]} '
              f'CH1:{d[1]}H/{d[2]}V vis={d[3]}  '
              f'CH2:{d[4]}H/{d[5]}V vis={d[6]}')
```

## 七、启用 ADC 实采模式

开机默认为内置测试波形（`WAVE_SRC_SINE`）；按 **BTN_RUN** 即可切换至 ADC 实采（`WAVE_SRC_ADC`），再次按下切回内置波形。运行时切换由 `main.c:handle_button()` 自动调用 `waveform_set_source()` 完成。

### 7.1 硬件

将示波器探头信号接至 `ADC_CH1_GPIO` 与 `ADC_CH2_GPIO`。

> ADC1 可用引脚：GPIO1~GPIO10（与 WiFi 兼容）  
> ADC2 引脚（GPIO11~GPIO20）在 WiFi 开启时**不可用**  
> ⚠ 默认 `ADC_CH1_GPIO = GPIO4` / `ADC_CH2_GPIO = GPIO5` 与 **ENC2/ENC3 引脚冲突**，启用前必须改为其他空闲 ADC1 引脚。

编辑 `main/adc_sampler.h`：
```c
#define ADC_CH1_GPIO   GPIO_NUM_?   // 改成未被占用的 ADC1 引脚
#define ADC_CH2_GPIO   GPIO_NUM_?
```

### 7.2 启用代码

在 `main/main.c` 中：

1. 文件顶部去掉 `/* #include "adc_sampler.h" */` 两侧注释；
2. `app_main()` 中（`button_init()` 之后）去掉下列两行两侧注释：
   ```c
   adc_sampler_init(adc_cb);
   adc_sampler_start();
   ```

无需再在初始化时手动调用 `waveform_set_source(WAVE_SRC_ADC)`——按键会做这件事。`adc_cb` 已在 `app_main` 上方定义，自动将采样数据送入波形模块。

ADC 量程：0~3.1V（11dB 衰减），分辨率 12bit（0~4095），采样帧率约 20fps。

## 十、常见问题

**Q：WiFi 连接后无日志输出**  
A：确认已连到 SSID `timer`，IP 为 `192.168.4.1`，端口 `4444`。如果连接建立后没有内容，等待系统下一次打印日志（心跳每 1 秒一次）。

**Q：LCD 无显示**  
A：检查 SPI 引脚接线；确认背光 GPIO40 被驱动为高电平；用示波器或逻辑分析仪确认 GPIO42（SCLK）有 40MHz 时钟信号。

**Q：编码器旋转无响应**  
A：检查 GPIO 接线，A/B 相是否接反（接反只影响方向，不影响响应）；用万用表确认 A/B 脚在旋转时有 0~3.3V 跳变。

**Q：UART 上位机收不到数据**  
A：确认 TX/RX 交叉连接（ESP32 TX → 上位机 RX）；确认波特率 115200；注意 GPIO43/44 是板载 CP2102N 调试串口，业务 UART1 在 GPIO17/18，二者不可混淆。

**Q：编译报错 `undefined reference to MACSTR`**  
A：在报错文件顶部加 `#include "esp_mac.h"`（ESP-IDF v5.5.3 不再隐式包含此头文件）。

**Q：编译时工具链被 Windows 拦截（错误代码 4551）**  
A：在 Windows 安全中心 → 病毒和威胁防护 → 排除项，添加 `C:\Espressif\` 目录，或通过组策略关闭智能应用控制。

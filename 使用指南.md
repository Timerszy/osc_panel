> 开发板：ESP32-S3-DevKitC-1-N16R8

> 作者：宋振豫（2205040336）

## 一、系统概述

本系统基于 ESP32-S3 实现一个示波器前面板控制器，通过 4 路旋钮编码器调节 CH1/CH2 的时基和电压档，通过 4 个按键切换工作模式，ST7789 LCD 实时显示示波器波形界面，并通过 UART 将所有面板操作事件上报给示波器主控。内置 WiFi SoftAP，可通过 TCP 无线查看全部运行日志。

```
┌─────────────────────────────────────────────┐
│  EC11 × 4         ESP32-S3           LCD    │
│  ENC1 CH1-H  ──→  ┌─────────┐  ←── ST7789  │
│  ENC2 CH1-V  ──→  │         │       240×240 │
│  ENC3 CH2-H  ──→  │  panel  │──→ UART 上位机│
│  ENC4 CH2-V  ──→  │  state  │              │
│                   │         │──→ WiFi 日志  │
│  BTN × 4     ──→  └─────────┘              │
│  LED × 3     ←──                           │
└─────────────────────────────────────────────┘
```

## 三、LCD 显示界面

LCD 全屏 240×240，布局如下：

```
┌────────────────────────────┐  ← y=0
│  H:100us  2:1V      IDLE  │  顶部信息栏（16px）
│  CH1时基  CH2电压   模式   │
├────────────────────────────┤  ← y=16
│  ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  │
│     ╔══════╗               │  CH1 绿色波形
│  ╔══╝      ╚══╗    ╔══     │
│══╝            ╚════╝   ══  │  CH2 青色波形
│                            │
│  8行 × 10列 刻度网格（224px）│
│  暗灰色网格线，中心线较亮   │
│                            │
└────────────────────────────┘  ← y=240
```

- **CH1**：绿色波形，振幅 ≈ 2.8 格
- **CH2**：青色波形，振幅 ≈ 2.0 格，相位偏移 60°
- 旋转 ENC1（CH1 时基）可改变 CH1 波形频率
- 旋转 ENC3（CH2 时基）可改变 CH2 波形频率
- 信息栏实时显示当前时基（H:）、CH2 电压档（2:）和工作模式

> 当前为正弦波占位模式（`WAVE_SRC_SINE`）；接入 ADC 后切换至 `WAVE_SRC_ADC` 即可显示真实采样波形。

## 四、操作说明

### 4.1 旋钮编码器

| 编码器 | 位置 | 功能 | 效果 |
|--------|------|------|------|
| ENC1 | CH1 左旋钮 | 时基（水平）| 顺时针→档位增大（更慢），逆时针→档位减小（更快），共 29 档：2ns ~ 5s |
| ENC2 | CH1 右旋钮 | 电压档（垂直）| 顺时针→档位增大（更大量程），逆时针→减小，共 13 档：1mV ~ 10V |
| ENC3 | CH2 左旋钮 | 时基（水平）| 同 ENC1，控制 CH2 |
| ENC4 | CH2 右旋钮 | 电压档（垂直）| 同 ENC2，控制 CH2 |

时基档位（29 档）：
```
2ns 5ns 10ns 20ns 50ns 100ns 200ns 500ns
1us 2us 5us 10us 20us 50us 100us 200us 500us
1ms 2ms 5ms 10ms 20ms 50ms 100ms 200ms 500ms
1s 2s 5s
```

电压档位（13 档）：
```
1mV 2mV 5mV 10mV 20mV 50mV 100mV 200mV 500mV 1V 2V 5V 10V
```

### 4.2 按键功能

| 按键 | GPIO | 功能 | 说明 |
|------|------|------|------|
| BTN_RUN | GPIO9 | 切换运行/示波器 | 当前为 RUN 则切到 SCOPE，否则切到 RUN |
| BTN_SCOPE | GPIO10 | 切换到示波器模式 | 强制进入 SCOPE 模式 |
| BTN_TDR | GPIO11 | 切换到 TDR 模式 | 强制进入 TDR 模式 |
| BTN_RESET | GPIO12 | 参数复位 | 所有通道恢复默认值（时基 100us，电压 1V），模式切换为 IDLE |

### 4.3 指示灯状态

| LED | 颜色 | 亮起条件 |
|-----|------|---------|
| LED_RUN | 绿 | MODE_RUN（运行采集中）|
| LED_SCOPE | 蓝 | MODE_SCOPE（示波器模式）|
| LED_TDR | 黄 | MODE_TDR（TDR 模式）|
| 全灭 | — | MODE_IDLE（空闲）|

## 五、WiFi 日志查看

系统上电后自动启动 WiFi SoftAP，所有 `ESP_LOGI/W/E` 日志同时输出至串口和 WiFi TCP 客户端。

### 5.1 连接步骤

1. 在手机/电脑的 WiFi 列表中找到 **`timer`**，点击连接（无密码）
2. 连接成功后设备 IP 为 `192.168.4.2`，ESP32-S3 的 IP 为 `192.168.4.1`

### 5.2 查看日志

**Python（推荐）：**
```python
import socket

s = socket.socket()
s.connect(('192.168.4.1', 4444))
print(s.recv(256).decode())  # 欢迎信息
while True:
    data = s.recv(4096)
    if not data:
        break
    print(data.decode(), end='')
```

**命令行（Linux/macOS）：**
```bash
nc 192.168.4.1 4444
```

**命令行（Windows PowerShell）：**
```powershell
$s = New-Object Net.Sockets.TcpClient('192.168.4.1', 4444)
$r = New-Object IO.StreamReader($s.GetStream())
while($true){ Write-Host $r.ReadLine() }
```

> 最多支持 4 路 TCP 同时连接。断开重连后日志继续输出，不会影响系统运行。

## 六、UART 通信协议

UART1，115200bps，8N1，TX=GPIO17，RX=GPIO18。

### 帧格式

```
[0xAA][0x55][TYPE][LEN][DATA...][CHECKSUM]
```

- `CHECKSUM = TYPE ^ LEN ^ DATA[0] ^ DATA[1] ^ ...`

### 事件类型

| TYPE | 名称 | DATA 格式 | 触发条件 |
|------|------|-----------|---------|
| 0x01 | ENCODER_EVENT | `[enc_id][dir]` | 每次旋钮转动 |
| 0x02 | BUTTON_EVENT | `[btn_id][state]` | 每次按键状态变化 |
| 0x03 | MODE_CHANGE | `[mode]` | 按键触发模式切换 |
| 0x04 | STATUS_REPORT | `[mode][ch1_h][ch1_v][ch2_h][ch2_v]` | 每秒心跳时 |
| 0x05 | HEARTBEAT | （空）| 每秒定时 |

### 枚举值

**enc_id：** 0=CH1-H，1=CH1-V，2=CH2-H，3=CH2-V  
**dir：** 0x01=顺时针，0xFF=逆时针  
**btn_id：** 0=RUN，1=SCOPE，2=TDR，3=RESET  
**state：** 0x01=按下，0x00=释放  
**mode：** 0=IDLE，1=SCOPE，2=TDR，3=RUN  

### 解帧示例（Python）

```python
import serial

ser = serial.Serial('COMX', 115200, timeout=1)

def read_frame(ser):
    while True:
        if ser.read(1) == b'\xaa' and ser.read(1) == b'\x55':
            ftype = ser.read(1)[0]
            flen  = ser.read(1)[0]
            data  = ser.read(flen)
            csum  = ser.read(1)[0]
            calc  = ftype ^ flen
            for b in data:
                calc ^= b
            if calc == csum:
                return ftype, data

while True:
    t, d = read_frame(ser)
    if t == 0x01:
        enc_names = ['CH1-H', 'CH1-V', 'CH2-H', 'CH2-V']
        print(f'ENC {enc_names[d[0]]}: {"CW" if d[1]==1 else "CCW"}')
    elif t == 0x04:
        modes = ['IDLE','SCOPE','TDR','RUN']
        print(f'STATUS mode={modes[d[0]]} CH1:{d[1]}H/{d[2]}V CH2:{d[3]}H/{d[4]}V')
```

## 七、切换到 ADC 实采模式

当前波形为正弦波占位，接入实际信号时按以下步骤切换：

### 7.1 硬件

将示波器探头信号接至 `ADC_CH1_GPIO`（默认 GPIO4）和 `ADC_CH2_GPIO`（默认 GPIO5）。

> ADC1 可用引脚：GPIO1~GPIO10（与 WiFi 兼容）  
> ADC2 引脚（GPIO11~GPIO20）在 WiFi 开启时**不可用**

如需修改 ADC 引脚，编辑 `main/adc_sampler.h`：
```c
#define ADC_CH1_GPIO   GPIO_NUM_4   // 改成实际引脚
#define ADC_CH2_GPIO   GPIO_NUM_5   // 改成实际引脚
```

### 7.2 代码

在 `main/main.c` 中取消以下注释即可，无需编写新代码：

**1) 文件顶部，取消 include 注释：**
```c
#include "adc_sampler.h"   // ← 去掉前面的 /* */
```

**2) `app_main()` 中，取消初始化块注释（位于 `button_init()` 之后）：**
```c
adc_sampler_init(adc_cb);
waveform_set_source(WAVE_SRC_ADC);
adc_sampler_start();
```

> 回调函数 `adc_cb` 已在 `app_main` 上方定义，会自动将 ADC 采样数据送入波形模块。

ADC 量程：0~3.1V（11dB 衰减），分辨率 12bit（0~4095），采样帧率约 20fps。

## 十、常见问题

**Q：WiFi 连接后无日志输出**  
A：确认已连到 SSID `timer`，IP 为 `192.168.4.1`，端口 `4444`。如果连接建立后没有内容，等待系统下一次打印日志（心跳每 1 秒一次）。

**Q：LCD 无显示**  
A：检查 SPI 引脚接线；确认背光 GPIO40 被驱动为高电平；用示波器或逻辑分析仪确认 GPIO42（SCLK）有 40MHz 时钟信号。

**Q：编码器旋转无响应**  
A：检查 GPIO 接线，A/B 相是否接反（接反只影响方向，不影响响应）；用万用表确认 A/B 脚在旋转时有 0~3.3V 跳变。

**Q：UART 上位机收不到数据**  
A：确认 TX/RX 交叉连接（ESP32 TX → 上位机 RX）；确认波特率 115200；注意 GPIO43/44 是板载 CP2102N 调试串口，业务 UART1 在 GPIO17/18，二者不可混淆。

**Q：编译报错 `undefined reference to MACSTR`**  
A：在报错文件顶部加 `#include "esp_mac.h"`（ESP-IDF v5.5.3 不再隐式包含此头文件）。

**Q：编译时工具链被 Windows 拦截（错误代码 4551）**  
A：在 Windows 安全中心 → 病毒和威胁防护 → 排除项，添加 `C:\Espressif\` 目录，或通过组策略关闭智能应用控制。

# 示波器面板显示调节控制系统 — 元器件选型清单

> 项目：示波器面板显示调节控制系统设计  
> 开发板：**ESP32-S3-DevKitC-1-N16R8**  
> 开发框架：ESP-IDF v5.5.3  
> 软件版本：含 WiFi SoftAP 日志服务、双通道波形显示、ADC 采样接口  

---

## 一、主控模块

| 器件名称 | 型号 | 数量 | 主要参数 | 选型理由 |
|---------|------|:----:|---------|---------|
| 主控开发板 | **ESP32-S3-DevKitC-1-N16R8** | 1 | 双核 Xtensa LX7 @ 240MHz，512KB SRAM，**16MB Octal SPI Flash**，**8MB Octal SPI PSRAM**，WiFi+BT5；板载 CP2102N USB-UART、USB-C、3.3V LDO（800mA）、WS2812B RGB LED；尺寸约 69×26mm | Espressif 官方开发板，原生 USB 烧录+调试；16MB Flash 与 8MB PSRAM 大幅扩展存储余量；引出全部可用 GPIO，排针间距 2.54mm 与面包板/PCB 直接兼容 |

> **注意事项：**  
> - **GPIO26~GPIO32**：SPI Flash 专用，禁止复用  
> - **GPIO33~GPIO37**：Octal PSRAM 专用（N16R8 的 R8 决定），禁止复用  
> - **GPIO19/GPIO20**：默认用于 USB-JTAG（D-/D+），如复用为普通 GPIO 则 JTAG 失效  
> - **GPIO43/GPIO44**：板载 CP2102N 占用（UART0 TX/RX），用于 PC 调试串口，勿与业务 UART 混用  
> - **GPIO48**：连接板载 WS2812B RGB LED，可用作额外状态指示

## 五、LCD 显示模块

| 器件名称 | 型号 | 数量 | 主要参数 | 选型理由 |
|---------|------|:----:|---------|---------|
| TFT 液晶屏 | **1.3" ST7789 240×240** | 1 | SPI 接口，RGB565，4线 SPI（MOSI/SCK/CS/DC），工作电压 3.3V，刷新率 ≥ 60fps@40MHz | 240×240 分辨率可显示 8×10 格示波器刻度网格 + 双通道波形；ST7789 驱动成熟，SPI 接线简单 |
| 背光控制 | GPIO 直接控制（板载 FET） | — | 由 LCD 模块内置 | 通过 GPIO40 控制，支持开关背光 |

> ⚠ GPIO33~GPIO37 在 N16R8 上被 Octal PSRAM 占用，LCD SPI 的 MOSI/SCLK/CS 已改至 GPIO41/42/47

## 六、通信接口模块

| 器件名称 | 型号 | 数量 | 主要参数 | 选型理由 |
|---------|------|:----:|---------|---------|
| UART 接口 | ESP32-S3 内置 UART1 | — | 115200bps，8N1，TX/RX 双线 | 向上位机/示波器主控发送面板事件（编码器、按键、状态帧）；帧格式：`[0xAA][0x55][TYPE][LEN][DATA][XOR]` |
| WiFi SoftAP 日志服务 | ESP32-S3 内置 WiFi | — | SoftAP：SSID=**timer**，无密码；TCP 服务器：192.168.4.1:4444，最多 4 路并发 | 所有 ESP_LOG 输出同时发往串口和所有已连接的 TCP 客户端，便于无线调试，无需额外硬件 |
| 电平转换（可选） | **SN74LVC1T45** | 2 | 1.2~5.5V 双向电平转换 | 若上位机为 5V UART 电平则需要；若上位机同为 3.3V 则可直连 |
| ESD 保护（可选） | **USBLC6-2SC6** | 1 | ±15kV ESD，双向 TVS | 保护 UART 引脚免受静电损坏，建议在连接外部系统时加装 |

> **GPIO 分配：** UART1_TX=**GPIO17**，UART1_RX=**GPIO18**  
> **注意：** GPIO43/GPIO44 为板载 CP2102N 调试串口（UART0），与业务 UART1（GPIO17/18）相互独立，无冲突

## 七、电源模块

> **DevKitC-1 已内置 3.3V LDO（800mA）及 CP2102N USB-UART 桥，无需另行设计主电源及烧录电路。**

| 器件名称 | 型号 | 数量 | 主要参数 | 说明 |
|---------|------|:----:|---------|------|
| 主电源（内置） | DevKitC-1 板载 LDO | — | USB-C 5V 输入，3.3V@800mA 输出 | 直接由开发板供电，外部器件从 3.3V 引脚取电 |
| 电源指示（内置） | 板载 LED | — | — | DevKitC-1 板载红色 PWR LED |

## 八、PCB 设计建议

> 使用 DevKitC-1 开发板时，PCB 主要承载外围器件 + 接插件，主控直接插排针或贴装。

| 项目 | 建议 |
|------|------|
| 板层 | 双层板，顶层走信号，底层铺地铜 |
| 主控安装 | DevKitC-1（69×26mm）通过 2.54mm 双排母针插接，预留板边空间供 USB-C 口维护 |
| 电源宽度 | 3.3V 分配线 ≥ 0.8mm（板载 LDO 输出电流充裕，走线阻抗需低） |
| 编码器走线 | AB 相等长，远离 SPI 时钟线（GPIO42） |
| SPI 走线 | MOSI/SCLK/CS（GPIO41/42/47）及 DC/RST（GPIO38/39）等长，线宽 0.3mm，串联 33Ω 匹配电阻 |
| UART 走线 | GPIO17/18 线宽 0.3mm，靠近板边引出接插件（GPIO43/44 为板载调试串口，勿引出至外部） |
| 去耦电容 | 在外设 VCC 引脚旁就近放置 100nF MLCC |
| USB 区域 | DevKitC-1 USB-C 口朝板边放置，预留维护空间；板上不另设 USB 电路 |
| 板尺寸 | 约 120×80mm（需容纳 DevKitC-1 的 69×26mm 插槽 + 外围器件布局） |

## 十、软件模块一览

| 模块文件 | 功能 |
|---------|------|
| `main.c` | 系统入口，任务创建，事件分发 |
| `encoder.c/h` | EC11 正交解码，产生编码器事件 |
| `button.c/h` | 按键消抖（FreeRTOS 定时器），产生按键事件 |
| `led.c/h` | 根据工作模式更新三色 LED |
| `lcd_st7789.c/h` | ST7789 SPI 底层驱动（fill/rect/char/hline/vline） |
| `waveform.c/h` | 示波器波形渲染（网格+双通道波形+信息栏），支持正弦波占位与 ADC 实采两种数据源 |
| `adc_sampler.c/h` | ADC1 oneshot 采样（GPIO可配置），可选自动校准，50ms 帧率 |
| `uart_comm.c/h` | 二进制帧协议，心跳/状态/事件上报 |
| `wifi_log.c/h` | SoftAP + TCP 日志服务，将 ESP_LOG 同步转发至串口和所有 TCP 客户端 |
| `panel_state.h` | 全局状态类型定义（模式、通道参数、事件结构体、档位表） |
| `pin_config.h` | 所有 GPIO/SPI/UART 引脚宏定义 |

## 十一、烧录命令参考

```bash
# 首次需全量清理（更换了 Flash/PSRAM 配置）
idf.py fullclean

# 编译
idf.py build

# 烧录（首次或全量）
idf.py -p COMX flash

# 仅烧录应用程序（加快速度）
idf.py -p COMX app-flash

# 监控串口输出（CP2102N，115200bps）
idf.py -p COMX monitor

# 通过 WiFi 查看日志（设备启动后）
# 先连接 WiFi: SSID=timer，无密码
python -c "import socket,sys; s=socket.socket(); s.connect(('192.168.4.1',4444)); [print(s.recv(256).decode(),end='') for _ in iter(int,1)]"
```

> 编译输出：`osc_panel.bin` 约 **320KB**（含 WiFi+波形模块），占用 16MB Flash 的不到 **2%**，存储余量充裕。
